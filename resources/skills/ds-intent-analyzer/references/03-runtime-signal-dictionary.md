# DS Intent Signal Dictionary v1

## Purpose

This document maps natural-language signals and contextual clues to:
- likely posture
- likely product realities
- likely pattern needs
- likely foundation directions
- caution notes

This is **not** a fixed conversion table.
It is a reading aid.

Keywords are only useful when interpreted inside context.

---

## How to use this dictionary

### Rule 1
Treat words as **signals**, not decisions.

### Rule 2
Prefer multi-signal patterns over single words.

### Rule 3
Use context and repeated jobs to resolve ambiguity.

### Rule 4
If wording conflicts with uploaded evidence, say so.

## Intent-level signals

### Analysis intent
Common wording:
- analyze
- break this down
- what does this suggest
- what kind of DS is this
- read this direction

Likely user need:
- interpret structure
- understand direction
- name constraints
- translate intuition into system logic

### Audit intent
Common wording:
- audit
- review
- inspect
- what is weak here
- what is missing
- what should I fix first

Likely user need:
- gap detection
- maturity review
- improvement priorities
- evidence-based critique

### Formation intent
Common wording:
- help me build
- recommend foundations
- propose tokens
- what should be the system direction
- what should we stabilize first

Likely user need:
- formation logic
- principle stack
- foundation priorities
- token proposal
- anti-overbuild guidance

### Comparison intent
Common wording:
- compare
- this feels like
- closer to X or Y
- what should I borrow
- what is the difference

Likely user need:
- reference interpretation
- careful borrowing
- cluster explanation
- trade-off framing

## Posture signal dictionary

### Trust-first
Common signals:
- trustworthy
- safe
- clear
- calm
- stable
- accessible
- plain
- formal
- government
- public service
- healthcare
- financial trust
- compliance

Likely pattern needs:
- guided forms
- error prevention
- clear status and feedback
- content-first hierarchy
- predictable navigation

Foundation tendencies:
- restrained semantic color
- high legibility
- moderate-to-generous spacing
- low-distraction motion

Main caution:
Do not confuse “minimal” with trust-first.

### Workflow-first
Common signals:
- efficient
- operational
- admin
- dashboard
- manage
- configure
- review
- approve
- monitor
- dense
- productivity
- task-heavy

Likely pattern needs:
- tables
- filters
- bulk actions
- forms
- status management
- validation
- empty/loading/error states

Foundation tendencies:
- scan-friendly typography
- tighter spacing control
- strong state color logic
- structured layout rhythm

Main caution:
A dashboard alone does not prove workflow-first.

### Workbench-tool surfaces
Common signals:
- workspace
- tuner
- analytics
- preview
- library
- export
- refine
- studio
- architect
- inspector
- workbench

Likely read:
- workflow-first tool with a persistent active object
- staged progression across multiple surfaces
- evaluate / refine loops that try to feel product-grade rather than purely decorative

Likely pattern needs:
- active-object persistence across views
- clear stage progression
- local action grouping near the current work object
- credible evaluation surfaces
- low-friction movement between tune, inspect, and apply steps

Foundation tendencies:
- dense but readable workspace structure
- strong hierarchy around the current object and next action
- restrained visual personality in service of tool clarity

Main caution:
Polished tool framing does not prove methodological rigor, trustworthy analytics, or real product maturity by itself.

### Accessibility-first
Common signals:
- inclusive
- accessible
- universal
- readable
- keyboard-friendly
- assistive
- compliant
- easy to follow

Likely pattern needs:
- strong focus and interaction states
- readable forms and content
- state clarity
- robust navigation and feedback

Foundation tendencies:
- strong contrast logic
- resilient type scale
- readable spacing
- reduced ambiguity in tokens

Main caution:
Accessibility-first is not automatically the same as Public Trust.

### Brand-first
Common signals:
- expressive
- premium
- iconic
- elegant
- emotional
- memorable
- luxury
- lifestyle
- differentiated
- refined

Likely pattern needs:
- controlled expressive moments
- stronger perception shaping
- tone-coherent interaction design

Foundation tendencies:
- deliberate color personality
- distinctive typography
- more expressive spacing rhythm
- stronger shape language
- intentional motion character

Main caution:
“Premium” can be shallow taste language.

### Platform-first
Common signals:
- reusable
- modular
- composable
- scalable
- multi-team
- design-code
- tokens
- theming
- primitives
- library
- extensible
- systemized

Likely pattern needs:
- reusable primitives
- semantic token layers
- theming structure
- adoption and contribution clarity

Foundation tendencies:
- disciplined naming
- layered token architecture
- consistency over novelty

Main caution:
Platform-like language is common now; reuse pressure must be real.

### Commerce-first
Common signals:
- merchant
- seller
- order
- product catalog
- inventory
- payment
- fulfillment
- payout
- revenue
- conversion
- business management

Likely pattern needs:
- product management
- order flows
- payment/settings states
- commercial alerts
- business filters and control patterns

Foundation tendencies:
- risk/value-aware state colors
- strong form clarity
- action-oriented layout rhythm

Main caution:
Do not classify from “ecommerce company” alone.

## Context clues that matter more than adjectives

These should carry more weight than style wording:
- who the primary user is
- what they repeatedly do
- what happens when they make mistakes
- how dense the work is
- whether trust/compliance matters
- whether identity expression is strategic
- whether the system must scale across many teams/products

---

## Contradiction patterns

### Pattern A — “Calm and premium” + dense admin tool
Likely read:
- Workflow-first or Trust-first foundation
- possible Brand-first accents only

### Pattern B — “Modern and minimal” + merchant operations
Likely read:
- Commerce-first with restrained expression

### Pattern C — “Scalable and reusable” + no cross-team reality
Likely read:
- maybe only early system ambition
- not enough for Platform-first

### Pattern D — “Accessible and clear” + strong emotional brand experience
Likely read:
- Brand-first with strong accessibility discipline

---

## Missing-signal prompts

When confidence is low, look for:
- who uses this most
- what tasks happen most often
- what mistakes are costly
- what screens/patterns dominate
- how much density is normal
- how much brand expression is strategically important
- whether this is shared across multiple teams/products
- whether a current system already exists

## Final rule

But context decides.
